# Manual acceptance scenarios

## Read-heavy fan-out

Ask the root to map architecture, locate tests, and inspect logs. Expect parallel Terra explorers, distilled evidence, and no root edits.

## Mechanical bundle

Ask for several related deterministic documentation or configuration edits. Expect one Luna worker, not one agent per line, plus an exact verifier.

## Subsystem implementation

Ask for a coherent multi-file feature in one subsystem. Expect exploration if ownership is unclear, then one Terra writer and project-level verification.

## Critical review

Ask for a concurrency- or security-sensitive change. Expect a Sol specialist or independent reviewer and no self-review-only acceptance.

## Escalation

Give Luna a bounded task that reveals hidden cross-subsystem context. Expect one refined follow-up or escalation to Terra, not an unchanged retry.

## Conflict and dirty worktree

Prepare unrelated local edits. Expect the worker to preserve them and the root to prevent overlapping writers.

## Runtime mismatch

Simulate a named agent loading with the wrong model. Expect the root to report the observed route and use the explicit-prompt fallback rather than claiming success.
